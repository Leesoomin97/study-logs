## 🧠 사전 지식
**기술 면접**이란 개발자 채용 과정에서 지원자의 기술적 역량을 평가하는 면접 유형이다.  
면접관은 지원자가 특정 기술 분야에 대한 지식, 문제 해결 능력, 코드 작성 능력을 갖추고 있는지 확인한다.  
이를 통해 회사는 실제 업무에 필요한 기술을 갖춘 인재를 선발할 수 있다.  

**기술적 질문 범위**
- 알고리즘 / 자료구조  
- 시스템 디자인  
- 특정 프로그래밍 언어 및 프레임워크  
- 실무 경험 및 문제 해결력

# 🖥️ (1) 컴퓨터 구조 & 운영체제

### 1. 프로세스의 정의
프로세스는 실행 중인 프로그램을 의미하며, **운영체제의 실행 단위**이다.  
프로세스는 독립된 메모리 공간(코드, 데이터, 스택, 힙)을 가지며 PCB를 통해 관리된다.

### 2. 스레드란?
프로세스 내의 **작업 단위**로, 같은 프로세스 내의 다른 스레드와 자원을 공유한다.

### 3. 커널이란?
운영체제의 핵심으로, **자원 보호 및 프로세스 관리**를 담당한다.  
페이징, 파일 시스템 관리 등도 수행한다.  
> 커널에 속하지 않은 OS 구성요소: UI

### 4. 프로세스가 입출력 작업을 만나면?
1. 시스템 콜로 입출력 요청  
2. 대기 상태로 전환  
3. 입출력 수행  
4. 완료 후 준비 상태로 전환  
5. CPU 재할당 후 실행 재개

### 5. CPU 스케줄링이란?
여러 프로세스에 CPU를 **효율적으로 배분**하는 운영체제의 핵심 기능.  
→ **FCFS, SJF, RR, Priority, Multilevel Queue** 등 알고리즘이 존재.

### 6. 부팅 과정
전원 인가 → BIOS 실행 → 부트로더 실행 → OS 로드 → 초기화 및 로그인

### 7. 페이징
메모리를 **페이지 단위로 분리**하여 관리하는 가상 메모리 기법.  
외부 단편화를 제거하고 프로세스 간 메모리 격리 가능.

### 8. 메모리가 크면?
동시 실행 프로세스 수 증가 → **멀티태스킹 능력 향상**

### 9. 시스템 콜
사용자 프로그램이 **커널 기능을 요청**할 수 있는 인터페이스.  
자원 접근은 반드시 시스템 콜을 통해야 한다.

### 10. 스레싱
페이지 폴트 과다로 CPU가 대부분의 시간을 페이지 교체에 소모하는 현상.  
→ **해결:** 물리 메모리 확장, 프로세스 수 조정, 교체 알고리즘 개선.

### 11. 파일 시스템
파일 및 디렉터리를 관리하는 커널의 일부.  
예: FAT, NTFS, APFS 등

### 12. 커널의 입출력 인터럽트 처리
1. 인터럽트 발생  
2. 상태 저장  
3. ISR(서비스 루틴) 호출 및 수행  
4. 상태 복원  
5. 재실행

### 13. 명령어 병렬 처리 기법
프로세서 성능 향상을 위한 기술.  
대표적: **파이프라이닝, 슈퍼스칼라, OOOE**

### 14. RAID
여러 디스크를 하나처럼 구성하여 **성능·안정성**을 높이는 기술.  
| 종류 | 특징 |
|------|------|
| RAID 0 | 스트라이핑, 빠름, 복구 불가 |
| RAID 1 | 미러링, 안정성 높음, 용량 절반 |
| RAID 5 | 패리티 분산, 성능+안정성 |
| RAID 6 | 패리티 2개, 안정성 최고 |

### 15. 레이스 컨디션과 동기화
여러 프로세스가 공유 자원에 동시에 접근할 때 발생하는 문제.  
→ **뮤텍스, 세마포어**를 이용해 동기화로 해결.

### 16. 파일 디스크립터
운영체제에서 **파일을 식별하는 정수값**.

### 17. 소켓
네트워크 프로세스 간 **데이터 송수신 인터페이스**.  
TCP(연결형), UDP(비연결형).

### 18. 하드 링크 vs 소프트 링크
| 구분 | 특징 |
|------|------|
| 하드링크 | 동일한 inode 공유, 원본 삭제 시 유효 |
| 소프트링크 | 다른 inode, 원본 삭제 시 무효화 |

### 19. 메모리 매핑
파일을 특정 메모리 영역과 연결하는 기법.  
→ **IPC(프로세스 간 통신)** 용도로 사용.

### 20. 시그널
프로세스 간 비동기 이벤트 전달 메커니즘.  
예: SIGINT, SIGKILL

# 🌐 (2) 네트워크

### 21. OSI 7계층
물리 → 데이터링크 → 네트워크 → 전송 → 세션 → 표현 → 응용  
각 계층은 독립적으로 기능 수행.

### 22. TCP vs UDP
| 구분 | TCP | UDP |
|------|-----|-----|
| 연결성 | 연결지향 | 비연결 |
| 신뢰성 | 높음 | 낮음 |
| 순서보장 | O | X |
| 속도 | 느림 | 빠름 |

### 23. ARP
IP ↔ MAC 주소 변환 프로토콜.

### 24. TCP 연결 수립 (3-Way Handshake)
1. SYN  
2. SYN+ACK  
3. ACK

### 25. 서브넷 마스크
IP 주소를 **네트워크/호스트 구간으로 분리**.

### 26. 쿠키
클라이언트 상태 정보를 저장하여 **세션 관리**에 활용.

### 27. DNS 리졸빙 과정
캐시 → 로컬 DNS → 루트 → TLD → 권한 서버 → IP 반환

### 28. 리다이렉션
요청 자원을 다른 경로로 이동시키는 응답.

### 29. HTTP 오류 코드
- **400대:** 클라이언트 오류  
- **500대:** 서버 오류  

### 30. 포트와 웰노운 포트
0~1023번: **표준 프로토콜 포트**

### 31. ICMP
네트워크 오류 보고 및 진단용 (ping 명령어).

### 32. 혼잡 제어
TCP는 **느린 시작(Slow Start)**, **혼잡 회피(CA)** 등 알고리즘을 통해 혼잡 방지.

### 33. 인터넷이 안 될 때 대처 순서
1. 물리 연결 확인  
2. 방화벽 확인  
3. 재부팅  
4. 외부 연결 테스트 (ping 8.8.8.8)  
5. 내부 연결 테스트 (ping 라우터)

### 34. 트래픽 과부하 시 대응
로드밸런서, 오토스케일링, 캐시, DB 이중화.

### 35. HTTP 캐시 검증
- `If-None-Match` (ETag)
- `If-Modified-Since` (Last-Modified)

### 36. Keep-Alive
하나의 TCP 연결로 여러 요청을 처리.

### 37. IPv4 → IPv6
IPv4 주소 부족 → 128bit 주소 IPv6 도입.  
(사설망/공유기 사용 덕에 IPv4 여전히 널리 사용 중)

### 38. UDP가 TCP보다 빠른 이유
연결 설정, 순서보장, 오류검증 과정이 없음.

### 39. 쿠키 보안 강화 방법
- `Secure` 플래그 (HTTPS만)
- `HttpOnly` 플래그 (JS 접근 차단)

### 40. PUT vs PATCH
| 메서드 | 의미 |
|--------|------|
| PUT | 전체 대체 |
| PATCH | 일부 수정 |

# 🗄️ (3) 데이터베이스

### 41. RDBMS vs NoSQL
RDBMS: 정형 데이터, SQL 기반  
NoSQL: 비정형 데이터, 유연한 스키마

### 42. 트랜잭션의 원자성
**모두 성공 or 모두 실패** → 데이터 일관성 유지

### 43. 인덱스
검색 속도 향상을 위한 자료구조.  
> 단, 저장공간/갱신 비용 증가.

### 44. Stored Procedure
DB 내에 저장된 **SQL 블록**으로 재사용 가능.

### 45. Join
| 종류 | 설명 |
|------|------|
| Inner | 교집합 |
| Left/Right Outer | 한쪽 테이블 기준 포함 |
| Full Outer | 전체 포함 |

### 46. HAVING 절
GROUP BY 이후 결과에 조건을 적용.

### 47. ON DELETE CASCADE
외래키 제약조건으로 **부모 삭제 시 자식 자동 삭제**.

### 48. View
SELECT 결과를 **가상의 테이블**로 제공.

### 49. ACID
원자성, 일관성, 격리성, 지속성.

### 50. SQL 언어 유형
| 유형 | 설명 |
|------|------|
| DDL | 구조 정의 (CREATE, DROP) |
| DML | 데이터 조작 (SELECT, INSERT) |
| DCL | 권한 제어 (GRANT, REVOKE) |
